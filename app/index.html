A peer-to-peer stack for code collaboration

Peer-to-peer

Open source development is fundamentally P2P. The Radicle stack runs on IPFS to enable fully peer-to-peer conversations with code.

Programmable

The Radicle collaboration apps bring issue management and patch proposals into the terminal. Use git-inspired commands to coordinate with peers and review your history offline.

Terminal-first

Apps are written in the radicle language, a Lisp dialect designed specifically for programming state machines for peer-to-peer replication. Tools and documentation for developing your own radicle applications coming soon!

Collaborate with Radicle

Manage your issues

rad issue brings issue management to your terminal.

Collaborate on code

rad diff is a way to collaborate on git repositories, completely peer-to-peer. Propose patches to any project and track your proposals.

Setup a project

rad project allows you to register your git projects on IPFS and connects your repo to other radicle applications, like rad issue and rad diff.

Radicle Stack

Let’s learn a little more about the technology stack
used to create this collaboration suite.


A radicle app is a state machine that specifies a complete model of all possible states and the allowable transitions between them.

State machines are programmed with the radicle language, a special Lisp dialect designed for P2P replication.

If IPFS is for P2P data, radicle is for P2P programs, where state updates are written as entries in a sequentially-hashed event log and the resulting radicle chain is synchronized with your peers over IPFS.

We’ll publish details on the complete stack soon, letting you fork and modify our applications, or make your own P2P apps.
