A peer-to-peer stack for code collaboration

Peer–to–peer

Open source development is fundamentally P2P. The radicle stack runs on IPFS to enable fully peer-to-peer conversations with code.

Programmable

Apps are written in the radicle language, a Lisp dialect designed specifically for programming state machines that replicate across a peer-to-peer network. Tools and documentation for developing your own radicle apps coming soon!

Terminal-first

Radicle's collaboration apps bring issue management and patch proposals into the terminal. Use git-inspired commands to coordinate with peers and manage contributions.

Collaborate with Radicle

Manage your issues

rad issue brings issue management to your terminal. Create, view, list and comment to your heart's content.

Collaborate on code

rad diff gives you the tools to collaborate on git repositories, completely P2P. Propose and track changes to a project you follow with a patch-based collaboration model.

Setup a project

rad project allows you to register your git projects on the radicle network and connect your repo to other radicle applications, like rad issues and rad diff.

Radicle Stack

Each radicle app is a peer-to-peer program that syncs with anyone who clones your open source project.

Taking inspiration from the original git model, where a single maintainer of a repo accepts patches, radicle apps are controlled by the owner of the project and updates can be submitted by any peer on the radicle network.

In the coming months we’ll be formalizing the technical stack underlying radicle and releasing documentation that will allow you to write your own apps using this same peer-to-peer infrastructure.
